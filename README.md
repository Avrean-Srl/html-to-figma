# HTML to Figma

A free Figma plugin that converts HTML + CSS into native Figma nodes - frames, text, images, Auto Layout, fills, gradients, shadows, borders - preserving the rendered layout. Optimized for Tailwind / shadcn / utility-first markup.

```
┌─────────────────────────┐   ┌─────────────────────────┐
│  Paste HTML in plugin   │ → │  Native Figma frames    │
│  (or drop a .html file) │   │  with Auto Layout and   │
│                         │   │  editable text          │
└─────────────────────────┘   └─────────────────────────┘
```

## Why this exists

Existing HTML-to-Figma tools are generalists. This plugin makes a deliberate trade: it targets the patterns Tailwind and shadcn produce, and it documents exactly what works and what doesn't (see [SUPPORT_MATRIX.md](SUPPORT_MATRIX.md)).

**Good for**: hand-off from code-first prototypes (cursor, v0, shadcn) to Figma. Importing a built component to review its design system, iterate on visuals, or build a Figma library off live HTML.

**Not good for**: pixel-perfect imports of arbitrary marketing sites. SVG illustrations with text labels. Form-heavy interfaces.

## Install

Search "HTML to Figma" in the Figma Community when published.

For local development, see [Development](#development) below.

## Use

1. Open the plugin from Plugins → HTML to Figma.
2. Either paste HTML into the textarea or drop a `.html` file onto the plugin window.
3. Pick a viewport width (320 / 768 / 1024 / 1440 / 1920). This drives how the hidden render container sizes the layout - it does not constrain the final Figma frame.
4. Click **Import to Figma**.

The imported tree appears at the center of your viewport and is selected, so you can immediately move it or paste it into a larger composition.

### What gets imported

A short summary - the full matrix lives in [SUPPORT_MATRIX.md](SUPPORT_MATRIX.md).

- `<div>`, `<section>`, `<article>`, etc. → `FrameNode`
- `<h1>`–`<h6>`, `<p>`, `<span>`, `<a>`, text content → `TextNode` (with font matched against your Figma fonts; falls back to Inter)
- `<img>` with `data:` or `https://` source → `RectangleNode` with image fill
- `<svg>` inline → `FrameNode` via `createNodeFromSvg`
- CSS `display: flex` → Auto Layout (direction, gap, padding, alignment, wrap)
- CSS gradients (linear, radial), shadows (drop and inset, multiple), uniform borders, blend modes, opacity, corner radius

### Failures are reported, not hidden

If an image URL is CORS-blocked or returns an unsupported format (SVG, WebP), the plugin renders a grey placeholder in the canvas and lists the failed URLs in the plugin UI after import. Layout is preserved.

## Development

Prerequisites: Node 20.19+ / 22.13+, pnpm 10+, Figma desktop.

```bash
pnpm install
pnpm watch              # rebuild on save
pnpm build              # production build
pnpm test               # 116 tests in real Chromium via Vitest browser mode
```

In Figma desktop: **Plugins → Development → Import plugin from manifest** → select the `manifest.json` at the repo root (regenerated by every build).

Architecture overview lives in [PROJECT.md](PROJECT.md). Locked design decisions and their rationale live in [DECISIONS.md](DECISIONS.md).
